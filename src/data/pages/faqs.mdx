## What is HyperUI?

HyperUI is a free, open-source library of Tailwind CSS components designed to accelerate your web development workflow.

## Is HyperUI free to use?

Absolutely! HyperUI is 100% free with no hidden costs.

## How do I use HyperUI components?

Using HyperUI is straightforward: browse our collection, find a component you need, copy the code, and paste it directly into your project.

## Do I need to install HyperUI?

No installation required. HyperUI is not a package. Just copy the component code you want to use.

## Is attribution required when using HyperUI?

No attribution is required, though it's always appreciated.

## How can I support the HyperUI project?

The easiest way to support HyperUI is by starring the project on GitHub. Every star helps!

## Where can I get help with HyperUI?

For support, please create an issue on the GitHub repository.

## How frequently is HyperUI updated?

Updates are regular with new components, collections, and features to keep the library fresh and relevant.

## Does HyperUI offer React components?

While the components are written in HTML, you can view and copy them as JSX for React projects.

## Are Vue components available in HyperUI?

Yes, the HTML components can be viewed and copied as Vue components to fit your project needs.

## Which Tailwind CSS version is HyperUI compatible with?

HyperUI currently uses Tailwind CSS v4.1.

## Why might components look different in my project?

Differences in appearance may be due to varying Tailwind CSS versions. Check your version first, and if issues persist, please open a GitHub issue.

## Are dark mode versions available?

Many components include dark mode variants, though not all components have this feature.

## Can I request new features or components?

Feature requests are welcome! Please submit them through GitHub issues.

## Can HyperUI be used in commercial projects?

Yes, HyperUI is free to use in any project, including commercial applications.

## Should I update my copied components when HyperUI updates?

Once you copy a component, it's yours to modify and maintain. We don't recommend trying to keep them synchronized with future HyperUI updates.
